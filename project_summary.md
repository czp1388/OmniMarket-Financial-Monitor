# 寰宇多市场金融监控系统 (OmniMarket Financial Monitor) - 项目总结

## 项目核心目标
开发一个支持多市场（A股、港股、牛熊证、外汇、加密货币、指数期货）、多时间周期（1分钟 - 1年）、集成行情监控、技术指标预警、流动性及资金管理的综合性金融分析平台。

## 功能模块规划

### 1. 数据源模块 (Data Source Module)
- 多市场接入：股票（沪深、港股、美股）、外汇、加密货币、指数/期货
- 多周期合成：从1分钟到年线的K线数据
- 数据存储：时序数据库（InfluxDB、TDengine）或金融数据库（DolphinDB）
- 数据清洗：自动处理缺失、错误数据

### 2. 行情图表模块 (Charting Module)
- 技术指标：MACD、KDJ、RSI、布林带、MA/EMA、成交量等
- 绘图工具：趋势线、斐波那契、水平线、文本标注
- 界面交互：多图表布局、图表联动、十字光标、缩放平移
- 多周期切换：无缝切换不同时间周期

### 3. 监控与预警模块 (Monitoring & Alerting Module)
- 条件类型：价格预警、技术指标预警、形态识别预警、成交量异动
- 预警方式：软件弹窗、声音、邮件、短信、Telegram/钉钉/webhook推送
- 预警管理：添加、编辑、启用/禁用预警条件

### 4. 流动性 & 资金管理模块 (Liquidity & Capital Management Module)
- 流动性监控：买卖盘口深度、订单簿动态、成交量、换手率
- 投资组合看板：仓位、盈亏、市值分布可视化
- 风险监控：单笔交易最大亏损、整体账户最大回撤、基于ATR的止损计算
- 模拟交易：实盘前的策略测试

### 5. 用户与系统管理模块 (User & System Admin Module)
- 用户登录和权限管理
- 个性化工作区布局保存
- 系统设置（数据源配置、通知渠道配置）

## 技术栈建议

### 后端
- 语言：Python（首选，生态丰富）或 Go（高并发性能）
- 框架：FastAPI（高性能，自动生成API文档）或 Django
- 数据存储：InfluxDB（时序数据）、PostgreSQL（关系型数据）、Redis（缓存）
- 实时通信：WebSocket（实时推送行情和警报）

### 前端
- 图表库：Lightweight Charts™ by TradingView（免费、高性能）或 ECharts
- 框架：React、Vue.js 或 Svelte（构建复杂SPA）

### 部署
- 服务器：Linux
- 容器化：Docker + Docker Compose
- 运维：7x24小时稳定运行，考虑多服务器、负载均衡

## 开发路线图

### 阶段一：最小可行产品 (MVP) - 基础监控
- 接入1-2个数据源（加密货币 + A股）
- 实现基础K线图表（使用Lightweight Charts）
- 实现最简单的价格预警
- 实现软件内弹窗提醒

### 阶段二：功能完善 - 成为分析工具
- 接入更多市场数据源
- 实现全部常见技术指标和绘图工具
- 实现技术指标预警
- 增加邮件、Telegram通知

### 阶段三：高级功能与风控 - 成为交易中枢
- 开发投资组合看板
- 实现流动性监控功能
- 开发风险预警功能
- （可选）接入券商API，实现半自动/全自动交易

## 面临的挑战与注意事项

1. **数据成本**：稳定、低延迟、全市场的实时数据昂贵，初期考虑免费或低成本API
2. **技术复杂度**：实时数据处理、海量数据存储、低延迟推送
3. **性能优化**：同时监控多个市场、品种、周期、预警条件
4. **法律合规**：数据使用合规性，实盘交易需格外谨慎
5. **团队需求**：需要前端、后端、量化金融知识的小团队协作

## 建议实施策略
从最熟悉的市场开始（如A股或加密货币），先实现单一市场的MVP，快速验证想法，再逐步扩展功能和市场。
